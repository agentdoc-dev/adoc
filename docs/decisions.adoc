::decision docs.markdown-prose-only-ingestion
status: accepted
decided_by: maintainer
owner: maintainer
evidence_ref: docs.adr-0023
--
Markdown sources are ingested as prose only: headings, paragraphs, lists, and code blocks become searchable prose, but no Knowledge Object is ever inferred from Markdown. Typing knowledge is a human act; a future migration tool may suggest candidates but never auto-type them.
::

::decision docs.mcp-patch-apply-opt-in
status: accepted
decided_by: maintainer
owner: maintainer
evidence_ref: docs.adr-0037
--
The MCP gateway registers adoc_patch_apply unconditionally, but the tool refuses to write unless the project's agentdoc.config.yaml sets mcp.patch_apply to enabled. Enabling agent writes is a deliberate, per-project human edit, never a default.
::

::decision docs.graph-status-slot-lifecycle-only
status: accepted
decided_by: maintainer
owner: maintainer
evidence_ref: docs.adr-0039
--
In adoc.graph.v4 the status slot carries lifecycle only; severity and trust travel as authored carrier fields and participate in the content hash. The additive-bump invariant is scoped explicitly: shapes untouched by the cleanup stay byte-identical, proven by goldens.
::

::decision docs.prose-ids-order-derived
status: accepted
decided_by: maintainer
owner: maintainer
evidence_ref: docs.adr-0040
--
Prose block identifiers are derived from document order, so insertions renumber downstream blocks and citation drift is accepted and documented. Only Knowledge Object IDs are stable citation targets; adoc why resolves Knowledge Objects, not prose blocks.
::

::decision docs.published-lists-guard-tested
status: accepted
decided_by: maintainer
owner: maintainer
evidence_ref: docs.adr-0041
--
Published MCP tool and kind lists are asserted against the code registry by guard tests using anchored lists and set-equality on names. Hand-maintained surface lists are retired; drift is a red test naming the file to fix.
::

::decision docs.pilot-thresholds-before-evidence
status: accepted
decided_by: maintainer
owner: maintainer
evidence_ref: docs.adr-0042
--
The dogfood pilot's un-gating thresholds for Markdown migration, composition, and governance were fixed in ADR-0042 before any pilot session ran, so gate decisions cite recorded evidence against pre-committed numbers rather than recollection.
::

::decision docs.canonical-source-identity
status: accepted
decided_by: maintainer
owner: maintainer
evidence_ref: docs.adr-0049
--
Source coordinates have three explicit roles: canonical physical paths read bytes, docs-root-relative identity paths derive page IDs, and validated project-relative logical paths populate diagnostics, graph spans, hashes, diffs, patches, and receipts. Project-bound Graph Artifacts identify agentdoc.config.yaml; standalone artifacts explicitly carry no repository identity.
::
