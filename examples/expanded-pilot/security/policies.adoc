# Security Policies @doc(security.policies)

Org-wide security policies. Policy authority comes from approvers and effective
dates, not from verification.

::policy security.production-db-access
status: active
owner: security-lead
approved_by: [security-lead, platform-lead]
effective_at: 2020-01-01
review_interval: 90d
related_to: security.audit.retention
--
Production database access requires an approved break-glass request and is logged for audit.
::

::claim security.audit.retention
status: verified
owner: security-lead
verified_at: 2020-01-01
source: internal audit records 2020-01-01
expires_at: 2024-01-01
--
Audit logs for production access are retained for at least 365 days.
::

::claim security.csrf-advisory
status: verified
owner: platform-security
verified_at: 2020-06-01
external_url: https://owasp.org/www-community/attacks/csrf
--
The OWASP CSRF advisory is the reference for cross-site request forgery mitigations applied across all API endpoints.
::
